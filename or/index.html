🔴 Oracle 학습 홈

기업 시장을 30년 넘게 지켜온 상용 RDBMS의 대명사, Oracle Database. SGA·PGA·백그라운드 프로세스부터 Undo/Redo, 옵티마이저까지 — 아키텍처를 그림과 애니메이션으로 꿰뚫습니다.

📚 챕터 12개 + 튜닝 팁 🎬 시각화 중심 학습 ✅ 완료한 챕터: 0 / 12

한눈에 보는 Oracle — SELECT 한 번의 여정

Oracle 아키텍처의 전체 그림을 애니메이션 하나로 먼저 느껴보세요. SELECT 한 문장이 클라이언트를 떠나 결과로 돌아오기까지의 여정입니다. 여기 등장하는 리스너, 서버 프로세스, SGA, 데이터파일이 앞으로 12개 챕터의 주인공들입니다.

클라이언트가 TCP 1521 포트로 접속을 요청합니다. 요청을 받는 것은 리스너(Listener) — 접속만 중개하고 쿼리는 처리하지 않는 "안내 데스크"입니다.

리스너가 이 세션을 전담할 dedicated 서버 프로세스를 만들어 연결을 넘겨줍니다. 이후 클라이언트는 이 프로세스와 직접 대화합니다.

SELECT 문장이 서버 프로세스에 도착해 파싱과 실행 계획 수립을 거칩니다(공유 풀의 커서 공유는 2장에서!).

서버 프로세스는 필요한 블록을 먼저 SGA의 Database Buffer Cache에서 찾습니다. 이번엔 캐시 미스!

캐시에 없으면 데이터파일에서 DB block을 읽어 버퍼 캐시에 올립니다. 다음번 같은 블록 요청은 메모리에서 바로 히트하겠죠.

블록에서 조건에 맞는 행을 골라 클라이언트에 돌려주면 여정 끝! 이 한 번의 왕복에 리스너(1장)·SGA(2장)·서버 프로세스(3장)의 개념이 모두 들어 있습니다.

가장 큰 사고방식 차이 하나만 기억하세요

PostgreSQL은 옛 버전을 테이블 안에 남기고(dead tuple → VACUUM이 청소), Oracle은 옛 버전을 undo라는 별도 공간에 복사해 둡니다. 그래서 PG에는 Bloat/VACUUM 고민이, Oracle에는 undo 크기/ORA-01555 고민이 생깁니다. 같은 MVCC, 다른 청구서입니다.
