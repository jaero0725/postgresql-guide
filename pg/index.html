한눈에 보는 PostgreSQL — SELECT 한 방의 여정

PostgreSQL의 전체 구조를 이 애니메이션 하나로 먼저 느껴보세요. SELECT 한 문장이 클라이언트를 떠나 결과로 돌아오기까지의 여정입니다. 여기 등장하는 등장인물(postmaster, backend, shared buffers, 디스크)이 앞으로 14개 챕터의 주인공들입니다.

클라이언트가 TCP 5432 포트로 접속을 요청합니다. 이 요청을 받는 것은 postmaster — 쿼리를 직접 처리하지 않는 "감독관" 프로세스입니다.

postmaster가 fork()로 backend 프로세스를 하나 만들어 줍니다. 연결 1개 = 프로세스 1개, PostgreSQL의 상징적인 설계입니다.

이후 클라이언트는 backend와 직접 대화합니다. SELECT 문장이 backend에 도착해 파싱·계획 수립을 거칩니다.

backend는 필요한 데이터 페이지를 먼저 shared buffers(공유 메모리 캐시)에서 찾습니다.

캐시에 없으면(cache miss) 디스크에서 8KB 페이지를 읽어 shared buffers에 올립니다. 다음번엔 캐시에서 바로 꺼내겠죠.

페이지에서 조건에 맞는 행을 골라 클라이언트에 돌려주면 여정 끝! 이 한 번의 왕복에 아키텍처(2장)·스토리지(4장)·플래너(6장)의 모든 개념이 들어 있습니다.
